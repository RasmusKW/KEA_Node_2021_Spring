API

API stands for Application Programming Interface. It contains a defined set of rules which allows program to talk to each other.
The API is created on the server side and allows clients to talk to it.

REST

REST stands for Representational State Transfer and is a software architectural style. It contains guidelines/rules that developers follow to create their API, so REST determines how APIs look. For a web service to be RESTful, it has to follow these guidelines.

RESTful APIs main practices comes down to four essential operations:

  • Receiving data in a covenient format.
  • Creating new data.
  • Updating data.
  • Deleting data.

A good rule to talk about is the one that states that you should be able to get a piece of data (resouce), when you link to a specific URL.
These URLs are called a Request and the data send back is called a Response.
The methods used to do these Request and Responses comes from HTTP which REST heavily relies on, see the HTTP Requests section for more info about each methods action. These methods are generally called CRUD (Create, Read, Update, Delete).

Callback Functions

A callback funtion is a function passed as an arguement to another function.
The smart thing about callback functions are that they help us deal with asynchronous programming. It does this by making sure that a function is not going to run before a task is completed but will run right after.

This next section will provide information about JavaScript Function Sequence and show relevant code snippets.
The function sequence in JavaScript are executed in the sequence they are called and not in the sequence they are defined.

For example (Example provided by w3schools) 

                                <p id="demo"> </p>

                                function myDisplayer(some) {
                                  document.getElementById("demo").innerHTML = some;
                                }
                                
                                function myFirst() {
                                  myDisplayer("Hello");
                                }
                                
                                function mySecond() {
                                  myDisplayer("Goodbye");
                                }
                                
                                myFirst();
                                mySecond();
This code will only end up displaying "Goodbye" and we therefore need a way to control our function sequence.

A way to do this could be by calling a function, save the result and then call another function to display the result of the function. 

                                <p id="demo"> </p>

                              function myDisplayer(some) {
                                document.getElementById("demo").innerHTML = some;
                              }
                              
                              function myMessageCompiler(txt1, txt2) {
                                let msg = txt1 + txt2;
                                return msg;
                              }
                              
                              let result = myMessageCompiler("Hello ", "Goodbye")
                              myDisplayer(result)
This would give us the result "Hello Goodbye". The problem with this solution is that we are using 2 function to display the result.

Another way to do it would be by letting a function call another function and call only that 1 function. 

                              <p id="demo"> </p>

                              function myDisplayer(some) {
                                document.getElementById("demo").innerHTML = some;
                              }
                              
                              function myMessageCompiler(txt1, txt2) {
                                let msg = txt1 + txt2;
                                myDisplayer(msg)
                              }
                              
                              myMessageCompiler("Hello ", "Goodbye");
This would also give us the result "Hello Goodbye". The problem with this solution is that there is no way to prevent the message compiler function from displaying the result.
This is where callback functions come in handy.

A callback funtion is a function passed as an arguement to another function. Here is an example following the message compiler example from above. 

                              <p id="demo"> </p>

                              function myDisplayer(some) {
                                document.getElementById("demo").innerHTML = some;
                              }
                              
                              function myMessageCompiler(txt1, txt2, myCallback) {
                                let msg = txt1 + txt2;
                                myCallback(msg)
                              }
                              
                              
                              myMessageCompiler("Hello ", "Goodbye", myDisplayer);

Here is a more simple example 

                            function print(callback) {  
                              callback();
                            }

Here is a callback example using the built in setTimeout method. 

                              const log = function() {  
                                console.log("This will be logged after 2 seconds");
                              }
                             
                              setTimeout(log, 2000);
Here the log function will be called after something happens (controlled by the setTimeout method), but not prior to that, which means it is an example of a callback function.
Lastly an example using ES6 arrow function 

                              setTimeout(() => { 
                                console.log("This will be logged after 2 seconds");
                            }, 2000);

A functional approach to Looping

Looping in JavaScript can be done by using functional approaches by using for example Map, Filter and Find.

Map

The map() method can be used to create a new array populated with the results provided by a function called on every element in the calling array.

Example 

                                const arr1 = [2,4,8,16]
                                const map1 = arr1.map(x => x/2)
                                console.log(map1);
Here we first define an array arr1 and here after use the map method on the array with the function x/2. This then calls that function on all the elements in the array and provides us with a new array with the result [ 1, 2, 4, 8 ]

Filter

The filter() method creates a new array with all elements that pass the test implmented by the given function.

Example 

                              const arr1 = ['this', 'is', 'the', 'filter', 'method']
                              const result = arr1.filter(arr1 => arr1.length > 3)
                              console.log(result);
Here we use the filter() method on the array which then tests for the length of each string element in the array to be larger than 3. With this we get the following result [ 'this', 'filter', 'method' ]

Find

The find() method returns the value of the first element in the provided array which satisfies the provided testing function. In the case that no values satisfy the testing function, undefined is returned.

Example 

                              const arr1 = [1,2,3,4];
                              const found = arr1.find(x => x > 1);
                              console.log(found);
Here we use the find() method on the array which uses the test function x > 1, checking each element if it's larger than 1.
The first value in the array that is larger than 1 is 2 and the result we get is therefore the value 2.

Type Coercion

Type coercion is the conversion of values from one data type to another (e.g. from number to string).
The difference between type corecion and conversion, is that coercion is implicit and conversion can be implicit or explicit.

Example 

                                const v1 = 3;
                                const v2 = '2';
                                let result = v1+v2;
                                console.log(result);
With this example, JavaScript automatically coerced the value 3 from a number into a string and then concatenated the two strings together.
This gives us the result 32.

If we want the compiler to coerce the string '2' into a number, we would have to explicitly convert the '2' to a number using the Number() method like this example. 

                                const v1 = 3;
                                const v2 = '2';
                                let result = v1+Number(v2);
                                console.log(result);
This Would give us the result 5.

Scopes

In JavaScript the scope determines the accessibility (visibility) of variables.
Functions in JavaScript each creates a new scope. The varibles defined inisde the function are not accessible (visible) outside the function, which means that they are Local. Variables defined outside of functions are accessible (visible) everywhere and is thereby Global.

This describes the two types of scopes there are in JavaScript.

  • Local Scope

  • Global Scope

Local variables are only recognized inside their functions, which means variables with the same name can be used in different functions.
The variables are created when a function starts and then deleted when the function is completed.

Variables declared outside a function are defined as Global.
A global variable has a global scope, which means all scripts and functions on a web page can access it.
Note: if you assign a value to variable that has not been declared, it will automatically become a Global variabel.

Example 

                                //This is a global variable which can be accessed from everywhere.
                                const myGlobal = "global";

                                //code here can not access the local variables declared in the myFunc() function.
                                //code here can on the other hand access var3 as it has been made Global automatically.
                                
                                function myFunc(){
                                  //these are Local Variables and can only be used inside this function
                                  const var1 = "x";
                                  const var2 = "y";
                                  //this variable has not been declared and is therefore automatically a Global varible.
                                  var3 = "q";

                                }

Strict Mode

Browsers that support it can run JavaScript in "Strict Mode".
In Strict Mode, undeclared variables are NOT automatically global. So in the example above var3 would not automatically be global.

URL Anatomy

URL
https://www.example.com:8080/path/?id=123#section-id
Key Function
Protocol Declares how your werb brower should communicate with the web server.
Subdomain A sub-division of the main domain name. Ex. blog.google.com is a sub-domain of google.com
Domain Unique reference that identifies a website on the internet.
Top-level Domain Always included in the domain name Ex. .com .org .dk etc.
Port Determines what port to establish connection to, rarely visible in the URL but always required.
HTTP uses port 80 and HTTPS (secure) uses port 443.
Path The path refers to a file or directory on the website e.g. /directory/file.php
Query String Usually found in dynamic pages (generated from a database or user generated), represented by a question mark followed by one or more parameters seperated by an ampersand (&) character. Ex. ?q=asdfad&ie=utf-8&aq=t
Fragment Identifier An internal page reference, begins with a hash (#) character followed by an identifier. It refers to a section within a web page.
